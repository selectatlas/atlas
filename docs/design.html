Atlas product language

A calm, quick-scanning system for finding the right person.

Blue makes intent obvious, lime marks meaningful match moments, and quiet surfaces keep the work moving. Clean, direct copy and Apple-inspired restraint keep the product trustworthy without losing Atlas character.

Source of truth: docs/design.md · Live app: /design-system · Tokens: src/app/globals.css

Overview

Expressive accents, quiet foundations

Atlas is a talent discovery workspace for hirers and creative talent. The system is compact, direct, and warm enough to make a people-first workflow feel human.

Voice & copy

Clean, direct, human

Apple-inspired editorial discipline: sentence case, verb-first actions, plain language. Hierarchy comes from weight and spacing — not shouting in ALL CAPS.

Core rules

  • Sentence case for labels, nav, section titles, and most UI copy.
  • Verb-first buttons: “View profile”, “Clear filters”, “Save changes”.
  • Select items and checkboxes; choose from menus and dropdowns.
  • Say what happened, then what to do — never error codes alone.
  • Use “you can” instead of “capability”, “leverage”, or “utilize”.
  • Reserve lime for match confidence — never as generic emphasis in copy.

Words to avoid

  • Abort, kill, invalid, uncheck, dialog box
  • Leverage, utilize, synergy, onboarding flow (say what it is)
  • ALL CAPS section labels (use muted label style instead)
  • “Error 422” or “Unprocessable entity” in user-facing text
  • “Click here” — name the destination or action

Do's and don'ts

The system is a guardrail

Make room for product-specific moments only when they carry meaning.

Do

  • Use semantic tokens like bg-primary and text-muted-foreground.
  • Reach for a shadcn primitive before a bespoke control.
  • Reserve lime for match confidence or an equally meaningful signal.
  • Write sentence-case labels and verb-first button copy.
  • Let spacing create hierarchy before adding borders or colour.
  • Blur only sticky chrome — keep cards and content opaque.

Don't

  • Introduce a new colour for a one-off card.
  • Use ALL CAPS eyebrows or filter section headers.
  • Mix oversized marketing treatments into compact work surfaces.
  • Hide the main action behind hover-only behaviour.
  • Turn every container into a pill or a floating shadow.
  • Show raw error codes or blame the user in alert copy.
